# FurCircle ‚Äî Phase 8 Spec: Monthly Wellness Progression

## Scope

Extend the monthly wellness plan so:
- Each `whatToDo` card has a step-detail view (Screen Two) with 2‚Äì4 sub-tasks, frequency badges, and optional video links
- Completed cards remain visible on Screen One with a strikethrough (not filtered out)
- When every card is complete ‚Üí current month is marked done ‚Üí next month's AI plan auto-generates

Depends on: Phase 2 (`logActivity`, `getHomeScreen`, Step Functions pipeline).

---

## What Changes

| Component | Current | After Phase 8 |
|-----------|---------|---------------|
| `whatToDo` item shape | `{ title, text, videoTopic }` | Adds `stepId` (slug) + `steps[]` (sub-tasks) |
| `getHomeScreen` actionSteps | Filters OUT completed items | Returns ALL items with `completed: boolean` flag |
| `logActivity` | Marks task complete, updates score | Also checks if month is done ‚Üí triggers next month |
| Step detail endpoint | Does not exist | `GET /dogs/{dogId}/plan/steps/{stepId}` |

---

## DynamoDB Changes

No new entities. Two attribute additions:

**Plan record** (`DOG#${dogId}` / `PLAN#${yyyy-mm}`):

| Attribute | Type | Description |
|-----------|------|-------------|
| `monthCompleted` | Boolean | Set to `true` when last task is logged |
| `completedAt` | String | ISO 8601 ‚Äî when month was completed |

`whatToDo` items gain two new fields (generated by Claude):

| Field | Type | Description |
|-------|------|-------------|
| `stepId` | String | Kebab-case slug, unique within plan (e.g. `basic-commands`) |
| `steps` | Array | 2‚Äì4 sub-task objects (see shape below) |

**Sub-task shape:**
```json
{
  "frequency": "Three times daily",
  "title": "Teach sit using a treat lure",
  "text": "Hold a treat above your dog's nose and slowly move it back...",
  "videoTopic": "lure shaping sit command"
}
```

`videoTopic` is optional. `videoTopic` is a search hint for the mobile app to surface a video ‚Äî not a URL.

## API Endpoints

### GET /dogs/{dogId}/plan/steps/{stepId}

Returns the full detail of one `whatToDo` card for Screen Two.

**Auth:** Cognito JWT (owner must match dog's `ownerId`)

**Path params:**
- `dogId` ‚Äî dog UUID
- `stepId` ‚Äî kebab slug from `whatToDo[].stepId`

**Response 200:**
```json
{
  "stepId": "basic-commands",
  "title": "Basic Commands This Month",
  "text": "Your Golden Retriever is at the peak learning window...",
  "completed": false,
  "steps": [
    {
      "frequency": "Three times daily",
      "title": "Teach sit using a treat lure",
      "text": "Hold a treat above your dog's nose and slowly move it back over their head...",
      "videoTopic": "lure shaping sit command"
    },
    {
      "frequency": "Daily",
      "title": "Reinforce name recognition",
      "text": "Say your dog's name once in a bright tone. Mark and reward the moment they look at you.",
      "videoTopic": "name recognition puppy"
    }
  ]
}
```

**Error responses:**

| Code | Error | Condition |
|------|-------|-----------|
| 403 | `FORBIDDEN` | `dog.ownerId !== userId` |
| 404 | `DOG_NOT_FOUND` | Dog record missing |
| 404 | `PLAN_NOT_FOUND` | No plan for current month |
| 404 | `STEP_NOT_FOUND` | `stepId` not found in plan (or plan predates Phase 8) |

### GET /home ‚Äî changes

Each item gains:
- `stepId: string` ‚Äî used by mobile to navigate to Screen Two
- `completed: boolean`

`plan` block gains:
- `completedCount: number`
- `totalTasks: number`
- `allComplete: boolean` ‚Äî convenience flag for month-complete banner

---

### POST /dogs/{dogId}/activities ‚Äî changes

Response gains optional field:
- `monthComplete: boolean` ‚Äî present and `true` only on the task that completes the month

## Claude Prompt Changes (`src/lib/claude.ts`)

New rules added to the existing prompt:

```
Each whatToDo item must include:
- "stepId": a unique kebab-case slug (2‚Äì5 words, no numbers), e.g. "basic-commands", "socialisation-walks"
- "steps": array of 2‚Äì4 sub-tasks, each with:
    - "frequency": how often (e.g. "Daily", "Three times daily", "Every session")
    - "title": 3‚Äì6 words
    - "text": 1‚Äì2 sentences of actionable instruction
    - "videoTopic": optional search phrase for a tutorial video

Return pure JSON only. No markdown.
```

**TypeScript interface update:**
```typescript
whatToDo: Array<{
  stepId: string;
  title: string;
  text: string;
  steps: Array<{
    frequency: string;
    title: string;
    text: string;
    videoTopic?: string;
  }>;
}>;
```

## `logActivity` Month-Completion Logic

After writing the ACTIVITY record and updating `categoryScores`, add:

```
1. Re-derive completedTexts for current month (Query ACTIVITY# SK or accumulate from plan)
2. If completedTexts.size === plan.whatToDo.length:
   a. UpdateItem plan: SET monthCompleted = true, completedAt = now
      ConditionExpression: attribute_not_exists(monthCompleted)   ‚Üê prevents race
   b. sfn.startExecution({ input: { dogId, month: nextMonth } })
   c. Include monthComplete: true in response
```

**IAM required:** Add `states:StartExecution` to `logActivity` function in `serverless.yml`.

**`validateInput` Lambda** (Step Functions first step) must accept optional `month` field and pass it through so `savePlan` stores the plan under `PLAN#${nextMonth}` instead of the current month.

## Backward Compatibility

Old plans in DynamoDB have no `stepId` or `steps[]`.

- `getHomeScreen`: `stepId` will be `undefined` on old plan items ‚Äî mobile must handle null `stepId` (disable tap navigation)
- `getPlanStep`: returns 404 `STEP_NOT_FOUND` for plans without `stepId` ‚Äî safe fallback

---

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| `logActivity` triggers Step Functions multiple times (race) | Medium | Medium | `ConditionExpression attribute_not_exists(monthCompleted)` ‚Äî only one write wins |
| Old plans missing `stepId` break Screen Two nav | High | Low | Mobile disables card tap when `stepId` is null; backend returns 404 gracefully |
| Claude returns duplicate `stepId` values | Low | Medium | Prompt rule: "unique within the plan"; unit test validates uniqueness |
| Next-month plan SK collision (plan already exists) | Low | Low | `savePlan` uses `attribute_not_exists(PK)` condition ‚Äî idempotent |
